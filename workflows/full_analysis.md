## 执行步骤

### Step 1: 需求梳理 (需求分析师)

**输入**: 业务方的分析需求

**输出**:
- 分析计划文档
- 数据需求清单
- 时间安排

**检查点**:
- [ ] 分析目标是否明确？
- [ ] 数据是否可获取？
- [ ] 时间是否合理？

### Step 2: 数据准备 (ETL 工程师)

**输入**: 数据需求清单

**输出**:
- 清洗后的数据集
- 数据质量报告
- SQL 脚本

**检查点**:
- [ ] 数据完整性是否满足？
- [ ] 数据质量是否合格？
- [ ] 计算逻辑是否正确？

### Step 3: 指标分析 (商业分析师)

**输入**: 清洗后的数据集

**输出**:
- 指标计算结果
- 趋势分析报告
- 异常点标注
- 业务洞察

**检查点**:
- [ ] 核心指标是否齐全？
- [ ] 趋势判断是否准确？
- [ ] 异常是否有解释？

### Step 4: 并行输出

#### 4a: 可视化 (可视化工程师)
- ECharts 图表配置
- 交互功能实现
- 响应式布局

#### 4b: 数据导出 (报告设计师)
- Excel 明细数据
- 数据透视表
- 条件格式

#### 4c: 策略建议 (策略顾问)
- 业务问题诊断
- 优化策略
- 行动计划

### Step 5: 报告整合 (报告设计师)

**输入**: 所有中间产出

**输出**:
- 完整分析报告 (HTML/PPT/Excel)
- 执行摘要
- 数据附录

## 时间估算

| 步骤 | 简单分析 | 中等分析 | 复杂分析 |
|------|----------|----------|----------|
| 需求梳理 | 0.5h | 1h | 2h |
| 数据准备 | 1h | 2h | 4h |
| 指标分析 | 1h | 2h | 4h |
| 可视化 | 1h | 2h | 3h |
| 报告整合 | 0.5h | 1h | 2h |
| **总计** | **4h** | **8h** | **15h** |

## 示例调用

```
对上月销售数据进行完整分析：
1. 分析 GMV、订单量、客单价等核心指标
2. 拆解到渠道、品类、用户群维度
3. 输出 HTML 交互报告 + Excel 数据表
4. 给出业务优化建议
```
